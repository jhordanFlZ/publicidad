# Edutainment Script Creator - Memory

## Formato y estilo de respuesta
- NO usar emojis en ningun output (instruccion critica del sistema)
- Siempre escribir en espanol (neutro latinoamericano) salvo que se pida otro idioma
- Usar estructura del system prompt exactamente: TEMA, PUBLICO OBJETIVO, CONCEPTO CREATIVO, GUION, etc.

## Patrones que funcionan bien

### Analogias de alto impacto para conceptos de codigo
- IF = portero de club / guardia de seguridad (universal, visual, graciosa sin conocimiento previo)
- Variables = cajas con etiquetas
- Funciones = recetas de cocina
- Bucles = disco rayado o Groundhog Day
- Arrays = filas en el supermercado

### Estructura de humor que funciona
- Contraste entre grandiosidad exagerada del personaje vs. lo mundano de su situacion real
- Error tecnico real (NullPointerException) expresado como reaccion fisica exagerada
- El espectador tecnico rie por reconocimiento; el general rie por la situacion. Doble audiencia, un solo chiste.

### CTAs que generan engagement real
- Pedir al usuario que APLIQUE el concepto en el comentario (no solo "dale like")
- Dar una plantilla de codigo simple que puedan completar con su vida cotidiana
- Ejemplo: "if (yo_codifico): print('true')"

## Estilos visuales pedidos hasta ahora
- Pixar 3D: paleta azul marino + morado neon + amarillo electrico. Personajes cuadrados con movimientos fluidos. Iluminacion cinematografica con rim light para heroe, neon sucio para comedia.

## Notas de produccion recurrentes a incluir siempre
- Indicar timing critico para gags visuales (cortes que deben ser instantaneos)
- Version por plataforma (TikTok 75s, YouTube Shorts 90s)
- Accesibilidad: el video debe funcionar en mute (subtitulos) y con audio
