# 16) Offer Ladder (Mandatory) — CTA by Stage

Guide all outbound conversations through this ladder. Do not jump to full implementation first.

## Offer 1 — 3-Bullet Diagnosis (Entry)

- **Promise:** Identify 3 concrete optimization points for one process.  
- **Delivery time:** 24–48 hours.  
- **Format:** 3 bullets (pain, root cause, quick recommendation).  
- **Best stage:** First reply / early conversation.  
- **Micro-CTA options:**  
  - "If I share a 3-bullet diagnosis for [process], would that be useful?"  
  - "Can I send you 3 practical improvements for [KPI]?"

## Offer 2 — Quick Win (Pilot)

- **Promise:** Implement one low-risk automation/integration with visible impact.  
- **Delivery time:** 3–7 business days.  
- **Scope examples:**  
  - WhatsApp reminder flow + status tagging  
  - Basic CRM sync from form or sheet  
  - One repetitive RPA task with logging  
- **Best stage:** After validation of need + authority.  
- **Micro-CTA options:**  
  - "If we implement one quick win this week, which process should we prioritize?"  
  - "Open to a mini pilot focused on [metric]?"

## Offer 3 — Implementation (Core Project)

- **Promise:** End-to-end modernization, automation, and integration roadmap + execution.  
- **Delivery time:** Depends on scope (phased).  
- **Scope:** Discovery, architecture, delivery milestones, support.  
- **Best stage:** After quick win success or clear strategic urgency.  
- **CTA options:**  
  - "Should we map a phased implementation plan for the next 90 days?"  
  - "Does it make sense to define scope, timeline, and ROI targets this week?"

## Stage Routing Rules

- New lead or cold reply -> Offer 1  
- Qualified and engaged lead -> Offer 2  
- Pilot validated or urgent enterprise need -> Offer 3

## Discipline rules

- Track output daily by channel (LinkedIn/email/WhatsApp-contextual).  
- If week finishes below 80% of target, run a 5-day recovery sprint at 30/day.
